Frequently Asked Questions (FAQ)

What is BugByte?

We intend to help users to keep track of bugs/defects in design documents, diagrams, formal specs, source files, test source files, binaries and data files.

How Do I Change My Password?

The admin would notify you about your username, password. If you do not know either your username or password, please contact the admin and they would assist you in creating a new password/username.

Is There A Support System Available If I Encounter Any Difficulties Using The Website?

For support in technical difficulties or bugs regarding the site, please contact the admin manually.

How Often Should I Update My Progress Or Report Bugs?

The frequency of reporting bugs of the project depends on the current progress of your project. If there are any defects or bugs that are found attempt to immediately post a report allowing others in the team to get notified of the issue.

How Can I Report A Bug Or Issue I've Encountered?

Once logged in, there is a button that allows you or your team members to enter the name/subject of the current bug, and a textbox to write a detailed summary of the issue. Furthermore, there would be timeline to show when each bug is discovered during the sprint, and which defects to prioritize.

Can I Update Or Edit Information I've Submitted Previously?

Currently, there is no way to edit previous bug posts, however if you noticed a mistake in your report, a new posted can be created to correct any mistakes.

Is There A Limit To The Number Of Team Members Who Can Access The Project Tracking?

There is no limit to the number of team members who can access the project tracking, as long as the admin creates an account for each team member, all members are able to access the website and report a bug in their project.

Are There Notifications Or Reminders For Upcoming Deadlines Or Critical Issues?

Yes. Once a bug report is posted, members when logged in will recieve a Notification of a bug. However, currently there are no notifications for any emergency deadlines or critical issues.
